# 网络小说爬虫与流行趋势分析系统 - 全系统说明文档

## 一、系统概述

### 1.1 项目名称
网络小说爬虫与流行趋势分析系统（Novel Spider & Trend Analysis System）

### 1.2 项目定位
本系统是一个面向中文网络小说的数据采集、清洗、分析与可视化展示的全栈应用。系统以 **17K小说网** 为目标数据源，通过自动化爬虫抓取小说元数据，经数据清洗、关键词提取、趋势统计等处理后，提供多维度的数据查询与可视化分析能力。

### 1.3 核心功能
- **自动化爬虫**：支持按分类抓取17K小说网的小说信息（书名、作者、分类、状态、字数、简介、更新时间等）
- **数据清洗**：HTML标签去除、文本规范化、分类映射、状态标准化、去重
- **关键词提取**：基于 jieba 分词，支持 TF-IDF 和 TextRank 两种算法
- **趋势分析**：按分类、状态、时间维度统计，计算增长率、活跃度、趋势评分
- **可视化展示**：柱状图、饼图、折线图、热力图、堆叠图等多种图表
- **后台管理**：爬虫任务管理、批次查看、系统健康检查、日志查看

## 六、核心业务流程

### 6.1 数据流水线（Data Pipeline）

```
步骤1：爬虫采集
  POST /api/process/crawl
  → crawl_service 创建/查找平台
  → HybridClient（优先Selenium，失败降级requests）
  → SeventeenKSpider 解析页面
  → 保存至 novel_raw 表

步骤2：数据清洗
  POST /api/process/clean
  → clean_service 读取 novel_raw
  → HTML标签去除、空白规范化
  → 分类映射（修真→仙侠，校园→都市等）
  → 状态标准化（连载中/已完结/暂停）
  → 字数解析、去重
  → 保存至 novel_info 表

步骤3：关键词提取
  POST /api/process/keywords
  → keyword_service 读取 novel_info
  → jieba 分词 + TF-IDF / TextRank 算法
  → 过滤停用词、噪声词、人名、数字
  → 保存至 keyword_stat 表

步骤4：趋势统计
  POST /api/process/trends
  → trend_service 读取 novel_info
  → 按分类/状态/时间维度聚合
  → 计算占比、增长率、活跃度、趋势评分
  → 保存至 trend_stat 表

步骤5：数据可视化
  GET /api/analysis/*
  → 查询各表数据
  → 返回JSON供ECharts渲染
```

### 6.2 爬虫工作流程

```
1. 用户发起抓取请求（单分类/全部分类/多分类）
2. crawl_service 初始化平台记录（如不存在则创建）
3. 生成批次号（格式：YYYYMMDD_HHMMSS）
4. SeventeenKSpider 开始抓取：
   a. 构造列表页URL（17K高级搜索页面）
   b. HybridClient 获取页面：
      - 优先使用 Selenium（无头Edge浏览器）处理JS渲染
      - 超时或失败时降级为 requests 直接请求
   c. BeautifulSoup4 解析HTML：
      - 列表页：提取小说详情链接
      - 详情页：提取书名、作者、分类、状态、字数、简介、更新时间
   d. 保存HTML快照（用于调试）
   e. 数据写入 novel_raw 表
5. 返回抓取结果（成功数量、失败数量、批次号）
```

### 6.3 数据清洗规则

| 处理项 | 规则 |
|--------|------|
| **HTML清理** | 去除所有HTML标签 |
| **空白规范化** | 合并连续空白、去除首尾空白 |
| **分类映射** | 修真→仙侠、校园→都市、网游→游戏 等 |
| **状态标准化** | 统一为"连载中"、"已完结"、"暂停" |
| **字数解析** | 从"约123万字"等格式中提取数字 |
| **去重策略** | 基于（书名 + 作者 + 平台）唯一约束 |
| **文本截断** | 简介过长时截断 |

### 6.4 关键词提取策略

| 环节 | 说明 |
|------|------|
| **分词引擎** | jieba |
| **算法** | TF-IDF（`extract_tags`）或 TextRank（`textrank`） |
| **来源字段** | 书名、简介 |
| **过滤规则** | 停用词表（40+词）、噪声词表（50+词，含平台名、通用词）、人名（100+姓氏）、数字 |
| **输出** | 每小说Top-K关键词（默认20） |


- **分层架构**：清晰的前后端分离、路由-服务-爬虫-数据四层设计
- **工厂模式**：Flask应用工厂，便于测试和扩展
- **Blueprint模块化**：4个独立路由模块，职责清晰
- **ORM抽象**：SQLAlchemy统一数据访问层

### 9.2 爬虫设计
- **混合采集策略**：Selenium + requests 双引擎，兼顾兼容性与性能
- **自动降级**：Selenium失败自动切换requests，提高鲁棒性
- **批次管理**：每次抓取生成唯一批次号，支持增量采集和历史对比
- **HTML快照**：保存页面快照，便于调试和回溯
- **反爬应对**：User-Agent伪装、请求延迟、重试机制

### 9.3 数据处理
- **完整ETL流水线**：Extract（爬虫）→ Transform（清洗）→ Load（入库）
- **智能分类映射**：自动归一化非标准分类名称
- **双重关键词算法**：TF-IDF + TextRank 可切换
- **多维度趋势分析**：分类、状态、时间三维统计
- **增长率计算**：支持跨批次对比，计算趋势评分

### 9.4 数据质量
- **去重机制**：基于（书名+作者+平台）唯一约束
- **数据验证**：清洗阶段校验字段完整性
- **原始数据保留**：novel_raw 保留原始数据，便于追溯

## 十二、常见问题预判

### Q1：为什么选择17K小说网作为数据源？
A：17K是国内主流网络文学平台之一，分类体系完善，数据结构规范，适合作为爬虫实践目标。

### Q2：爬虫如何应对反爬机制？
A：系统采用三重策略：①User-Agent伪装模拟真实浏览器；②请求间设置延迟（默认3秒）；③Selenium无头浏览器处理JS渲染页面，失败自动降级为requests。

### Q3：关键词提取的准确性如何保证？
A：系统使用jieba分词的TF-IDF和TextRank两种算法，并建立了包含40+停用词、50+噪声词、100+人名的过滤词表，有效过滤无效词汇。

### Q4：系统如何保证数据质量？
A：通过三层保障：①原始数据保留（novel_raw）；②清洗阶段规范化处理；③数据库唯一约束去重。

### Q5：趋势分析的计算逻辑是什么？
A：系统按批次统计各分类/状态的小说数量、更新数量、活跃数量，计算占比（share_value）和环比增长率（growth_rate），综合得出趋势评分（trend_score）。

### Q6：系统是否支持扩展到其他网站？
A：支持。爬虫模块采用抽象基类设计（BaseSpider），只需实现具体解析逻辑即可适配新网站。配置中的CRAWLER_BASE_URL也可灵活修改。
